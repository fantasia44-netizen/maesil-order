# AutoTool 통합시스템 — 코드 리뷰 요청서

> **작성일**: 2026-03-23
> **목적**: 외부 AI 코드 리뷰 + 아키텍처 평가 + 개선점 도출
> **대상**: 이유식 기업(쿡대디/배마마) ERP/WMS 통합시스템

## 1. 프로젝트 개요

이유식 제조/판매 기업의 **주문→재고→생산→출고→배송→정산** 전체 운영을 관리하는 통합 ERP/WMS 시스템.

- **스택**: Flask 3.1 + Supabase (PostgreSQL) + Bootstrap 5 + Render 배포
- **운영 시작**: 2025년 11월 (약 4개월 운영)
- **개발 방식**: Claude AI 에이전트 기반 1인 개발

### 계층 구조
```
HTTP 요청
  → auth.py (인증/RBAC/세션/속도제한)
  → blueprints/*.py (라우트, 입력 검증)
  → services/*.py (비즈니스 로직)
  → core/*.py (데이터 무결성 검증)
  → db_supabase.py (DB 계층)
  → Supabase REST API
```

### 외부 연동
```
마켓플레이스 API:
  ├── 네이버 스마트스토어 (OAuth2 + bcrypt HMAC)
  ├── 쿠팡 Wing (HMAC-SHA256 per-request)
  ├── Cafe24 (OAuth2 refresh token)
  ├── 11번가 (API Key) — 스켈레톤
  ├── 옥션/G마켓 ESM (JWT) — 스켈레톤
  └── 카카오 톡스토어 (OAuth) — 스켈레톤

택배사 API:
  └── CJ대한통운 표준 API V3.9.3 (1Day Token)
      ├── 고객번호 A: 30494329 (외부송장)
      └── 고객번호 B: B133030971 (넥스원출고)

금융 API:
  ├── CODEF (은행/카드 거래내역)
  └── 팝빌 (세금계산서) — 보류, 홈택스 엑셀 사용

## 5. 데이터 흐름

### 주문 → 출고 → 배송 플로우
```
[마켓 API] ─→ fetch_orders() ─→ api_orders (원본 저장)
                                    ↓
                              api_orders_to_excel_df()
                                    ↓
                              OrderProcessor.run()
                                    ↓
                    ┌───────────────┼───────────────┐
                    ↓               ↓               ↓
            order_transactions  order_shipping   재고차감
            (주문/상품 정보)    (배송/송장 정보)  (stock_ledger)
                                    ↓
                    ┌───────────────┼───────────────┐
                    ↓               ↓               ↓
              CJ API 예약접수   CJ 엑셀 업로드   수동 처리
              (채번+등록)       (매칭 서비스)
                    ↓               ↓
                    └───────┬───────┘
                            ↓
                    order_shipping.invoice_no 갱신
                            ↓
                    마켓 push_invoices()
                    (네이버/쿠팡/카페24 API)
                            ↓
                    shipping_status = '발송'
                            ↓
                    배송상태 추적 (폴링)
```

```

                                          ↓
                                    매출-입금 매칭
                                    (matching_service)
                                          ↓
                              ┌───────────┼───────────┐
                              ↓           ↓           ↓
                        bank_matching  정산 대사   P&L 반영
